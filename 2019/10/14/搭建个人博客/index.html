搭建个人博客

发表于 2019-10-14 分类于 搭建个人博客

这是我写的第一篇博客，不得不说，还是有点小激动的，废话不多说，开始进入正题！

1. 前言

1.1 准备工作

​ 在开始之前，你需要有github账号（github官网），安装node.js(node.js官网)，因为node.js安装包里面带有npm，所以也不需要另外安装，还有git客户端（git官网）。所有都准备好之后就可以开始了。

2. 搭建GitHub博客

2.1 创建仓库

​ 新建一个名为用户名.github.io的仓库（必须是用户名，其他名称无效，想知道原因的童鞋可以去了解一下github pages），将来你将代码托管到仓库后就可以通过https://用户名.github.io去访问你的博客了。

2.2绑定域名

​ 不绑定域名也是可以的，默认是上面的用户名.github.io来访问，如果你想拥有不一样的域名，也是可以的哦。

​ 这里我就不多说了，感兴趣的可以去这里链接绑定一下（这里以阿里云绑定域名为例了）

3. 配置SSH key

​ 说实话，我也不知道配置这个的作用（俺也不知道，俺也不敢问）。看别人的博客上说是为因为提交代码的时候需要github权限，直接使用用户名和密码不安全，使用这个可以解决本地和服务器的连接问题。

​ 首先可以检测下本地是否有存在的ssh密钥

​ 如果提示：no such file or directory 就说明是第一次使用git.然后你就需要生成下ssh key。

​ 然后连续3次回车，最终会生成一个文件在用户目录下，具体的生成目录到时候也会提示出来，如下图所示:

​ 然后打开id_rsa.pub文件，复制里面的内容，然后打开github主页，进入个人设置->SSH and GPG keys ->New SSH key,如下图所示:

​ 最后可以测试测试下是否成功。

​ 如果提示Are you sure you want to continue connecting (yes/no)?，输入yes,然后可以看到

​ 配置完后最最最后在配置下github用户名和邮箱（这个俺也不知道是干嘛用的）

4. 使用hexo写博客

4.1 hexo简介

​ hexo是一个简单、快速、强大的基于 Github Pages 的博客发布工具，支持Markdown格式，有众多优秀插件和主题。（[官网地址](http://hexo.io
github: https://github.com/hexojs/hexo)）

4.2 原理

​ 由于github pages存放的都是静态文件，博客存放的不只是文章内容，还有文章列表、分类、标签、翻页等动态内容，假如每次写完一篇文章都要手动更新博文目录和相关链接信息，相信谁都会疯掉，所以hexo所做的就是将这些md文件都放在本地，每次写完文章后调用写好的命令来批量完成相关页面的生成，然后再将有改动的页面提交到github。
